## 2. 기술 스택

| 항목 | 선택 | 이유 |
|------|------|------|
| 프레임워크 | **Next.js 14 (App Router)** | GIA 사이트와 동일 스택 → 코드 재활용, 학습 비용 0 |
| 스타일 | Tailwind CSS | 동일 |
| 다국어 | next-intl + `[locale]` 세그먼트 | `/kr/~`, `/en/~` URL 기반 라우팅 |
| 배포 | Vercel | 무료 플랜으로 충분, 서브도메인 설정 불필요 |
| 이미지 처리 | Canvas API + browser-image-compression | 100% 클라이언트 사이드 → 서버비 0 |
| 결제 | (추후) Toss Payments | Pro 플랜 추가 시 |

### Next.js를 선택한 이유

다른 선택지와 비교:

| | Next.js 14 | Astro | SvelteKit | Vite + React (SPA) |
|---|---|---|---|---|
| SEO | 최상 | 최상 | 상 | 하 ❌ |
| 번들 크기 | 중 | 최소 | 최소 | 중 |
| i18n | next-intl | 별도 설정 | 별도 설정 | 복잡 |
| GIA 코드 재활용 | ✅ 최상 | 불가 | 불가 | 불가 |
| 생태계/레퍼런스 | 최상 | 상 | 중 | 최상 |
| Vercel 배포 | ✅ 네이티브 | 됨 | 됨 | 됨 |

**결론**: Astro가 정적 사이트로는 이론상 더 가볍지만, 이미지 Crop·PDF 변환 등 인터랙티브 툴이 많아지면 Islands 아키텍처가 오히려 복잡해짐. 한국어 SEO 레퍼런스도 거의 없음. 툴 대부분이 클라이언트 연산이므로 App Router Server Components 장점은 크게 못 쓰지만, GIA 스택 재활용 이점이 더 큼.

### 아키텍처 방향

- 대부분의 툴 페이지는 `'use client'` 중심으로 단순하게
- 각 툴이 독립 URL 보유 → SEO 키워드 타겟팅 가능
- 이미지 처리는 서버 없이 브라우저에서 처리

## 3. URL 구조

### 확정된 구조

```
tooltoolz.com/                          → 루트 → 브라우저 언어 감지 후 /kr 또는 /en 리다이렉트
tooltoolz.com/kr/                       → 한국어 홈 (툴 목록)
tooltoolz.com/en/                       → 영어 홈 (툴 목록)

tooltoolz.com/kr/tools/char-count       → 글자수 세기 (한국어)
tooltoolz.com/en/tools/char-count       → 글자수 세기 (영어)
tooltoolz.com/kr/tools/json-format      → JSON Formatter (한국어)
tooltoolz.com/kr/tools/img-compress     → 이미지 압축 (한국어)
...
```

### 언어 전환 방식
- 루트(`/`) 접근 시 `Accept-Language` 헤더로 자동 감지 → `ko` 포함이면 `/kr`, 아니면 `/en`
- 헤더의 언어 토글로 `/kr` ↔ `/en` 수동 전환 가능
- 서브도메인(`kr.tooltoolz.com`) 방식은 사용하지 않음 → Vercel 추가 설정 불필요

### Next.js App Router 라우팅

```
app/
├── page.tsx                            ← redirect → /kr or /en (미들웨어 처리)
└── [locale]/                           ← 'kr' | 'en'
    ├── layout.tsx
    ├── page.tsx                        ← 홈
    └── tools/
        ├── char-count/page.tsx
        ├── json-format/page.tsx
        └── img-compress/page.tsx
```

### middleware.ts (루트 리다이렉트)

```ts
import { NextRequest, NextResponse } from 'next/server'

export function middleware(request: NextRequest) {
  const { pathname } = request.nextUrl
  if (pathname === '/') {
    const lang = request.headers.get('accept-language') ?? ''
    const locale = lang.startsWith('ko') ? 'kr' : 'en'
    return NextResponse.redirect(new URL(`/${locale}`, request.url))
  }
}

export const config = {
  matcher: ['/'],
}
```

- 카카오 애드핏 또는 구글 애드센스 — 툴 페이지 하단 배너

### 중기
- Pro 플랜 — 배치 처리, 이미지 100장 압축 등 대량 기능
- API 판매 — 개발자 타겟

### 카테고리별 예상 RPM
| 카테고리 | 예상 RPM |
|----------|----------|
| 개발자 툴 | 높음 ($3~8) |

| 일반 계산기 | 중 ($1~3) |
| 생성기/게임 | 낮음 ($0.5~2) |

## 7. SEO 전략

- 각 툴이 독립 URL 보유 (`/kr/tools/[slug]`, `/en/tools/[slug]`)
- 페이지 타이틀: `"글자수 세기 온라인 무료 - tooltoolz"` 형태
- 한국어 특화 키워드 타겟: `"글자수 세기"`, `"JSON 포맷터"`, `"이미지 용량 줄이기"` 등
- 툴 페이지마다 사용법 설명 섹션 추가 → 롱테일 키워드 흡수
- 언어별 hreflang 태그 설정: `<link rel="alternate" hreflang="ko" href="https://tooltoolz.com/kr/tools/[slug]" />`

## 8. 디자인 방향

- **컨셉**: 군더더기 없이 툴만 남긴 깔끔한 UI
- **색상**: 카테고리별 색상 구분 (텍스트=파랑, 이미지=초록, 계산기=앰버, 틈새=퍼플, 생성기=코랄)
- **광고 없음** 강조 → 신뢰도 선점 (나중에 붙이더라도 UX 해치지 않는 위치)
- **다크모드** 지원
- **네비게이션**: 카테고리 탭 + 검색바
